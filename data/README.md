# `data/` ‚Äî data dictionary

Every data file in this repo, what it holds, and its schema. JSONL/NPZ/CSV can't carry inline
comments, so this file is the authoritative description.

## Layout & the one hard constraint

```
data/
  <raw / env-loaded datasets>          # stay at root (see below)
  README.md                            # this file
  derived/
    paths/    path & harvest data      (merged_best_paths, pilot_results, *_index)
    labels/   training labels/splits   (dot_archive, dot_splits)
    dot/      d-o-t crafting outputs    (percentiles, baseline, anchors, trap-set)
```

**Env-loader contract (why raw files stay at root):** `envs/ac_s.py` opens datasets as
`open(f"data/{initial_states_file}.txt")` ‚Äî a bare stem like `"AC19_extended"` resolves to
`data/AC19_extended.txt`. So every `.txt` loaded by stem is **anchored to `data/` root** and must not
move into a subfolder (it would break training/beam). `greedy_all.txt` is *generated* but lives at
root for this reason ‚Äî the lone exception to "generated files go under `derived/`".

**Encoding:** relators are `xXyY` strings (`X`=x‚Åª¬π, `Y`=y‚Åª¬π). The canonical join key is
`"<r1>|<r2>"` from `scripts/lib/canon.py::canon_key` (rotation+inverse+swap class, pair ordered
length-then-lex under `Y<y<X<x`). The env int8 literal encodes `x‚Üí1, X‚Üí-1, y‚Üí2, Y‚Üí-2`, zero-padded to
`2¬∑24 = 48` ints per presentation.
