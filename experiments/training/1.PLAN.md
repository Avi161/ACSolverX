## 1. Objective & success criteria

Train `f(presentation) → d-o-t` (predicted min S-moves to the trivial presentation), to be used as a **search heuristic ranking sibling moves**, replacing relator length.

Because the use is ranking, success is **ordering quality**, not just absolute error:

1. **Beat the length-only baseline** on test **MAE and Spearman**.
2. **Beat it by a larger margin on the hard band** (high d-o-t), where length carries least information — this is the whole premise.
3. **Honest hard-hump generalization** measured on the frozen `hard_test` (never trained on).
4. (Downstream, later phase) f-guided search solves ≥ GS-Sub at matched node budget, or shortens beam's mean path (12.39).

---

## 2. Data & featurization

- **Model input:** int8 length-48 presentation, built from canonical `(r1,r2)` via `canon.strs_to_presentation_literal` (L=24). This is the DRT's native input.
  - **Do NOT canonicalize the model input as a feature** — rely on the DRT's cyclic relative-position embeddings (paper: DRT > hard-canonical preprocessing by ~25 solves). Canonical form is used **only** as the label/dedup key, which is already baked into the archive.
- **Target:** `min_dot` per class. Train on `log1p(d-o-t)` (tames the heavy tail to 349); report MAE in original scale via `expm1`.
- **Scalar features (14):** relator lengths, exponent sums, letter counts, total/diff/max/min — for the GBM baseline only.
- **Censored rows (4,954):** unsolved initials, no point label. Handling in §5.
- **Provenance:** each class records its contributing source(s) (greedy/beam) → source weighting (§6).

---

## 3. Baselines (the bar to beat) — `experiments/training/baselines.`*

1. **Global mean** — error floor.
2. **Length-only** — mean d-o-t per total relator length, fit on train. *This is the heuristic we are replacing; the project exists to beat it.*
3. **Scalar GBM** — HistGradientBoosting / XGBoost on the 14 scalar features. Tests whether cheap hand features already beat length (and by how much), establishing a strong non-deep baseline.

Report MAE + Spearman on **test and hard_test** for all three. GBM runs on **CPU** → can be a local script (needs `sklearn`/`xgboost` installed in a py3.10+ env or the notebook), or a cell in the notebook. **Decision in §12.**

---

## 4. Model

Reuse `network.py:RelativeDualRingActorCritic` **unchanged** (the dual-ring transformer: per-relator self-attention + cross-attention + cyclic relative-position embeddings; ~140k params, trains fast on any GPU).

Two head options:

- **(A) Value-output reuse [v1 default].** Train the existing scalar **value head** to regress `log1p(d-o-t)`. Zero new module code. The actor head is still computed but unused (receives no gradient). **Enables warm-start for free** (same module → exact param-tree match with `610model`).
- **(B) Fresh regression head on the trunk [v2].** `RelativeDualRingBlock × N → masked-mean pool → MLP → 1`, defined as a new flax module reusing the imported blocks. Needed when we add auxiliary/uncertainty heads (§7).

**Warm-start (default on):** restore `ppo_checkpoints/610model` params (two-stage Orbax, params only). The trunk already encodes AC-relevant structure; the PPO value *scale* differs from d-o-t but the head adapts under Huber. **Fallback: fresh init** (wrapped in try/except) so the notebook always trains.

Start with **(A)+warm-start** for a fast, robust first number; move to **(B)** for aux heads.

---

## 5. Losses

- **Solved (labelled):** **Huber on `log1p(d-o-t)`**. log1p for the tail; Huber for robustness to loose upper-bound labels.
- **Censored (unsolved):** one-sided **hinge** `max(0, log1p(B) − pred)` — penalize only predictions *below* the budget bound `B`, leaving the model free to predict arbitrarily high (AK(3) "is allowed to be hard"). Requires a per-row `B` (the search budget reached). **v1: train solved-only**; add censored in v2 once `B` is decided (§12).
- **Ranking term (v2, optional):** pairwise margin loss on sibling states (same-path neighbours) — directly optimizes the ordering search relies on.

---

## 6. Training procedure

- **Optimizer:** adamw, lr 3e-4, small weight decay; cosine or plateau LR decay.
- **Batch** 256; **epochs** ~80–150 with **early-stop on val MAE** (patience ~15).
- **Source weighting:** cap/down-weight over-represented sources so neither swamps the other (PLAN.md §6).
- **Regularization:** dropout in blocks if the train/val gap widens (37k examples vs 140k params → moderate overfit risk).
- **Determinism:** fixed seed; everything logged to a run dir under `experiments/training/runs/<ts>/`.
- **Checkpoint:** save best-val params (msgpack) to Drive.

---

## 7. Auxiliary heads (v2, head option B)

Cheap, share the trunk:

- **Solvable-within-budget** (binary) — uses solved + censored.
- **Predicted max-intermediate-length** (expected hump size).
- **Uncertainty** (deep ensemble or MC-dropout) → **OOD abstention** in search (trust `f` only where confident; else fall back to policy/length).

---

## 8. Evaluation (the real test, not just MAE)

- **MAE** (original scale) + **Spearman** on test.
- **Pairwise sibling-ordering accuracy:** for on-path neighbour pairs, fraction where `f` orders them correctly — the metric that predicts search quality. Needs neighbour generation (`s_move_np` / `get_neighbors`).
- **Per-difficulty-band MAE + calibration** (easy ≤5 / mid 6–15 / hard >15) — the average is dominated by easy cases and hides hard-hump usefulness.
- **Frozen `hard_test`** — the honest generalization number to the hard hump.
- `**counterexample_eval` (the real bar):** the frozen AK(n)-family fold
(`experiments/eda+data_collection/DATA_DISTRIBUTION_PLAN.md §5`), never trained. Report what fraction of `AK(3)…AK(8)` the model ranks as **hard** (pred ≥
the `B_hard` floor) — i.e. does it refuse to call the look-alikes easy? This is the metric that
the whole anchor effort exists to move.
- **AK(n) trap-rate (search-time, ties to PLAN §8):** when `f` guides search, load
`data/ak_trap_set.json` and, for every expanded state, test canonical-key membership. Per run
log whether the trajectory **fell into an AK(n) basin**, which `AK(n)`, and at what depth;
aggregate a **trap-rate** (% of runs trapped) + a per-`AK(n)` histogram. Directly measures the
paper's "algorithms just reduce hard cases back to AK(3)" failure and whether `f` escapes it or
re-finds it. The trap-set + counter live in one place so this metric and the downstream
f-guided search (`experiments/PLAN.md §8`) count against the identical definition.
- **Novel-basin discovery (open-world, the other half):** flagged traps **not** in the trap-set
are *newly-discovered* local minima — logged to a resumable registry and ranked by how many
runs hit them (`experiments/PLAN.md §8`). Recurring novel basins are **promoted** into
`ak_trap_set.json` and injected as new censored anchors on the next data build, so the hardness
signal grows as the model meets new problems. Track the count of distinct novel basins found per
eval run as a reported output (not just the closed-world AK(n) rate).
- **Calibration plot** (predicted vs best-known), **all vs baselines**.

## 10. Risks & mitigations (from PLAN.md §1)


| risk                                              | mitigation                                                                             |
| ------------------------------------------------- | -------------------------------------------------------------------------------------- |
| **Distribution shift** (killed Caltech's attempt) | trained on greedy+beam **mix** (done — beam supplies detours); DAgger in a later phase |
| **Upper-bound labels**                            | min-aggregation (done) + censored loss + never punish under-prediction                 |
| **Ranking ≠ MAE**                                 | report Spearman + pairwise sibling ordering; consider ranking loss                     |
| **Overfit**                                       | frozen hard hold-out, val early-stop, weight decay/dropout                             |
| **Source imbalance**                              | source weighting                                                                       |
| **Warm-start scale mismatch** (PPO value ≠ d-o-t) | Huber + the head adapts; fallback fresh init                                           |

## 11. Concrete step order

1. **Commit** `dot_archive.jsonl` + `dot_splits.npz` to `test/eda` (notebook inputs).
2. **Baselines** (length-only + GBM) → record the bar on test + hard_test.
3. `**train_dot_model.ipynb`** — head (A) + warm-start → train → eval vs baseline on test + hard_test. **This is v1.**
4. **Iterate** → head (B), aux heads, censored loss, ranking term (v2).
5. **Wire `f` into search** (PLAN.md §3) and benchmark — separate phase.
