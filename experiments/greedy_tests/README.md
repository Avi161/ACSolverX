# greedy_tests

Test suite for the greedy AC-search pipeline: `experiments/search/greedy_baseline.py`
(normal + `HIGH_SPEEDUP` solvers) and `experiments/run_baseline.py` (the runner).

## Running it

```bash
# default tier -- run after ANY change to the pipeline (~3 min; +30-60s cold numba JIT)
.venv/bin/python3 -m pytest experiments/greedy_tests -q

# full tier -- run before any push or result claim (~15-20 min)
.venv/bin/python3 -m pytest experiments/greedy_tests -q --runslow
```

`slow` tests (golden sweeps, multiprocessing, deep parity) are skipped unless
`--runslow` is passed. Temporary files go to `.pytest_tmp/` inside the repo,
never `/tmp`.

## How it is built

Three layers, so that a bug in one cannot hide a bug in another.

**1. `spec/` -- a general-`n` reference, written from the definitions.**
Pure Python, no numba. Words are tuples of signed ints, so `n_gen` is a free
parameter. Minimal rotation is brute-forced rather than Booth's; reduction comes
straight from the definition. Agreement with the solver is therefore evidence,
not a restatement of the same code.

**2. `spec/invariants.py` -- the independent oracle.**
The exponent-sum matrix is a quantity the solver never computes. A substitution
move is an elementary row operation on it, so `det` is preserved exactly;
canonicalisation only negates or permutes rows, so `abs(det)` is preserved; the
trivial presentation has `M = I`. Hence every state reachable from an AC-trivial
presentation has `abs(det) == 1`. All 640 solved and all 261 unsolved
representatives satisfy this, and so does every neighbour the real expander
emits. An expander that invents an unsound move fails here even if the spec has
the same bug.

**3. `adapters.py` -- the seam.**
Each solver is a `SolverAdapter` that declares which presentations it `supports`
and returns a normalised `SearchStats`. `test_solver_contract.py` iterates
`ALL_ADAPTERS` x fixtures and skips unsupported pairs.

## Adding the stable-AC solver

The code today is 2-generator/2-relator at every layer: the state is an `(n, 2)`
bool array with one bit for "which generator", `expand_node_nj` loops
`for target in range(1, 3)`, and `pack_key` packs exactly two segments around one
`0x00` separator.

When the stabilized solver lands:

1. write an adapter, and append it to `ALL_ADAPTERS`;
2. that is all.

The contract suite, the invariant tests and the packed-key sort-order property
already run at `n_gen = 3` today against `SpecAdapter`, so they are live
acceptance criteria rather than dead skips. `test_solver_contract.py::
test_the_numba_adapters_are_two_generator_only_today` is the one test that must
be updated, and it exists to tell you so.

Two things to know before writing that solver:

* **Definition 2.1 does not generalise verbatim.** It replaces `r_i` by
  `rot_{k1}(r_i) . rot_{k2}(r_{3-i})^j`, and `r_{3-i}` only means "the other one"
  when there are exactly two relators. At `n_rel > 2` the move needs a real
  source index. `spec/moves.py` defines it as `Move(i, j, s, k1, k2)`.
* **The codebase's `j` is the invert sign, not a relator selector.** The
  literature's general-`n` `j` is the selector. Do not conflate them.

## Golden baseline

`golden/greedy_golden.json` is regenerated by

```bash
.venv/bin/python3 -m experiments.greedy_tests.tools.regen_golden
```

**A golden failure is a result change, not a stale fixture.** The search is
deterministic, so a number that moved means something altered the search. Find
out what, and say so in the commit message, before regenerating.

Only deterministic fields are stored. `min_relator` and `max_relator` are picked
with `min()`/`max()` over a `set` in the normal solver, so their tie-breaks
follow `PYTHONHASHSEED` and differ between processes *and* between the normal and
heavy solvers. Their lengths never do, and only the lengths are recorded.

## Deliberate non-assertions

* **AK(3) and the MMS02 instance are never asserted solvable.** Their plain
  AC-triviality is open. They are fixtures for "does not crash, respects the
  budget, preserves `abs(det) == 1`" only. A test asserting a solve would be
  asserting an open conjecture and would pass on luck.
* **The contract never compares node counts across adapters.** A correct stable
  solver may explore in a different order. Exact-trace parity is a property of
  the normal/heavy pair, which are two implementations of one search.
* **Change of variables is covered as a mechanism, not as a procedure.** Its
  "isolate" step needs AC1-3 conjugation and its Lemma-11 removal is proved with
  an unbounded number of substitutions. `test_stable_ac.py` covers AC4, the
  general move and AC5 preserving the invariants -- which is what a solver built
  on the idea must respect.
