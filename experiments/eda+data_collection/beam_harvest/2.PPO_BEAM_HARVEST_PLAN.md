# PPO+Beam Harvest Plan — better d-o-t labels for the supervised model

## Goal

Run the trained PPO policy + beam search over **every** presentation in
`data/all_presentations_len_8_to_19_GS_solved_copy2.csv` (17,635 rows) to produce
**step-by-step trivialization paths in the same format as that CSV**, so we can:

1. **Shorten** paths greedy already solved (shorter path ⇒ smaller, *closer-to-true*
   distance-until-trivialization (d-o-t) label), and
2. **Newly solve** some of the 4,954 presentations greedy *failed* (greedy is
   best-first by total relator length; a learned policy explores differently and may
   reach states greedy never expanded).

The output feeds a single merged, deduplicated **min-d-o-t label table** keyed by
canonical state — the training target for the regressor.

> **Honesty guardrail (carried from the EDA):** beam search optimizes policy
> log-probability and stops at the *first* beam to hit the trivial presentation — it
> is **not** a guaranteed length-minimizer. So a beam path can be longer than
> greedy's. We therefore take the **minimum** d-o-t per canonical state over
> {greedy, beam, multi-seed beam}. "Beam gives better labels" is the expected
> outcome, measured per-state, not assumed.

## One correction to the stated assumption (important)

You said: *"the canonical representative we are using is the same one in the paper and
what they use for ppo beam, so we can reuse the code."*

- **Reuse the search code: yes.** `beam/beam_search.py` runs unchanged. The PPO env
  (`ACS`) is correct and we feed it our presentations directly.
- **But the PPO/beam env is NOT in the lab's canonical form.** Internally the env
  state is the flat `int8` array, and after each move it Booth-rotates **only the
  just-modified relator** (numeric `int8` order, no inverse-minimization, no pair
  reordering). That is a *partial* normalization — **not** `canonical_pair_nj`
  (greedy's canonicalizer) and **not** the paper's Definition E.1.
- **Consequence:** the search is fine, but to make harvested paths "just like the
  CSV" *and* to merge labels across greedy+beam **by state**, we must canonicalize
  every output state through `canonical_pair_nj` — the exact post-processing already
  validated in `experiments/eda.ipynb` **P2-11** (100% neighbor coverage there).
  This is an **added output step**, not a change to the solver.

So: reuse the solver, canonicalize the outputs. The canonical key for all
joining/dedup is `canonical_pair_nj` (reduce → lex-min over rotations+inverse → pair
ordered length-then-lex, alphabet `y⁻¹ < y < x⁻¹ < x`).

## Pipeline overview

```
greedy CSV (17,635 rows: r1,r2,…)
        │  Phase 1: convert + index
        ▼
data/greedy_all.txt  (+ data/derived/paths/greedy_all_index.csv)      env-readable initial states
        │  Phase 2: beam_search.py (chunked, Colab GPU, 610model policy)
        ▼
beam_out/greedy_<a>_<b>.csv   (packed action paths per SOLVED presentation)
        │  Phase 3: replay each packed path → record every state → canonicalize
        ▼
data/beam_paths.csv           (same schema as greedy CSV: r1,r2,Nodes,Path Length,Path)
        │  Phase 4: union with greedy on-path states, key by canonical_pair_nj, MIN d-o-t
        ▼
data/dot_labels.parquet       (canonical_key → dot_min, per-source dot, provenance)
```

Reusable canonical functions get extracted **once** into `scripts/lib/canon.py` (port of
`reduce_relator_nj`, `canonical_pair_nj`, `str_to_arr`, `arr_to_str` from
`greedy_search.ipynb`; `@njit` made optional). Both Phase 3 and Phase 4 import it.
This is also the `canon()` module the broader roadmap (`CLAUDE_ROUGH_PLAN.md`) needs.

- Read the greedy CSV. For **all 17,635 rows** (solved *and* greedy-unsolved — the
  unsolved ones are the highest-value new-solve targets; they still have `r1,r2`),

  - char→int map `x→1, X→-1, y→2, Y→-2`; pad each relator to length 24 with `0`;
    concatenate → 48-int list; write `repr(list)` per line.

- Write:
  - `data/greedy_all.txt` — the env-readable initial-states file (stem `greedy_all`).
  - `data/derived/paths/greedy_all_index.csv` — `line_idx, r1, r2, greedy_solved, greedy_path_length`
    so beam results join back to the original rows and to the greedy d-o-t labels.
- **Validation:** assert line count == 17,635; assert no relator contains `0` before
  padding; round-trip a few lines back to strings and `assert == original (r1,r2)`.

**New module: `scripts/lib/canon.py`** (extract the validated P2-11 port)

- `reduce_relator_nj`, `canonical_pair_nj`, `str_to_arr`, `arr_to_str` lifted verbatim
  from `greedy_search.ipynb` (strip `@njit`, or keep numba if installed).
- Add `env_state_to_strs(x_int8, max_length=24) -> (r1_str, r2_str)`: split the 48-int
  array at `max_length`, drop padding zeros, map ints→`xXyY`. (This is the bridge from
  env space to the string interchange format the greedy functions expect.)
- Add `canon_key(r1_str, r2_str) -> (key_str, total_len)` = canonicalize via
  `canonical_pair_nj(reduce(...))`, same as P2-11's `cpkey`.
- **Validation:** import it and re-run the P2-11 equivalence check (stored greedy
  states are 1:1 with `canonical_pair_nj` classes) to confirm the extraction matches
  the notebook (25,209 == 25,209, zero d-o-t disagreement).

## Phase 2 — Beam search over `greedy_all` (paper config, chunked, on GPU)

Use `beam/beam_search.py` **unchanged**, with the pretrained policy and the
**paper's Appendix I ("Hybrid Beam Search Details") config**:

- `--beam_width 16384` (paper B = 16,384)
- `--max_steps 150` (paper T = 150; also the search horizon — beams die on truncation
  at `max_steps`, so this must be the horizon, not the training NUM_STEPS)
- `--alpha 0` (paper scores by action log-prob only; **value head not used**)
- **Gumbel exploration across 5 seeds** (paper reports min/max/mean over 5 seeds). The
  paper's "5 seeds" are 5 *independently trained agents*; we have one checkpoint, so we
  adapt to **5 beam RNG seeds on 610model** and take the **min** solving path per
  presentation. Seed 0 = deterministic (`--temperature 0 --temp_end 0`); seeds 1–4 =
  Gumbel-explored.
- **The one knob the paper does not pin: the Gumbel temperature value/schedule.**
  Default for the explore seeds: `--temperature 1.0 --temp_end 0.0` (linear decay,
  explore-early/exploit-late). Flagged below as overridable.

```bash
# one chunk × one seed; sweep seed in {0..4} and chunk-start across [0,17635)
SEED=0; TEMP=0.0          # seed 0 = deterministic; seeds 1..4 use TEMP=1.0
python beam/beam_search.py \
  --ckpt_path 610model \
  --dataset greedy_all \
  --training_dataset AC19_extended \
  --start 0 --end 2000 \
  --beam_width 16384 \
  --max_steps 150 \
  --alpha 0.0 \
  --temperature $TEMP --temp_end 0.0 \
  --seed $SEED \
  --out_csv beam_out/greedy_s${SEED}_0_2000.csv
```

- `--dataset greedy_all` → searches our presentations. `--training_dataset
  AC19_extended` only sizes the dummy `solve_data` for restore; the search never uses
  it (610model's own `solve_data` may shape-mismatch and is restored best-effort — a
  warning there is expected and harmless).
- **Chunking:** the outer loop is a plain Python `for` over presentations; split the
  range with `--start/--end` and write one CSV per `(seed, chunk)` into `beam_out/`.
  Chunks/seeds are independent and resumable — safe across multiple Colab sessions.
- **Output per file:** rows with `presentation_idx, solved, path_length, path`
  (packed action ints), `path_decoded`. `path` is what Phase 3 replays. Phase 3
  collapses the 5 seeds by keeping the shortest solving path per `presentation_idx`.

**Compute reality — this is the expensive part.** The paper ran B = 16,384 only on the
~1190-presentation MS benchmark. We are proposing it on **17,635 presentations × 5
seeds ≈ 88k beam searches**, each up to 150 steps of a DRT forward pass over 16,384
beams (plus a re-sorted `B·T = 2.4M`-entry visited buffer per step). That is **far**
beyond the paper's beam budget — realistically many GPU-hours to days. Mitigations, in
order of preference:
  1. **Pilot first (Phase 0)** at full B=16,384 on ~200 presentations to get a real
     per-presentation wall-clock and solve rate, then extrapolate before committing.
  2. **Width ladder:** our presentations are len 10–17 (easier than the MS hard
     regime); most likely solve at far smaller B. Run a cheap pass first (e.g.
     B=1,024, seed 0) to clear the easy majority, then re-run **only the unsolved
     remainder** at the full paper B=16,384 × 5 seeds. Same labels, a fraction of the
     compute. (We keep the paper config as the *ceiling* applied where it matters.)
  3. **Chunk across sessions** (zero new code).
  4. **Optional `jax.vmap` batched beam** (documented CLAUDE.md lesson) — big speedup,
     new code, must pass the equivalence guard (batched temp-0 solve length == serial).
     Build only if the chunked run is too slow.

## Phase 3 — Replay packed paths → step-by-step states (CSV-format)

**New script: `scripts/harvest_paths.py`**

`beam_search.py` stores only the action sequence, so we replay to recover states.

- **First, collapse seeds:** across all `beam_out/greedy_s{0..4}_*.csv`, keep for each
  `presentation_idx` the **shortest** solving path (ties → seed 0 / deterministic).
  This is the "min over 5 seeds" that yields the shorter-than-greedy labels.
- For each **solved** (collapsed) beam row, build `ACS(n_gen=2, max_length=24,
  max_steps_in_episode=path_length, initial_states_file="greedy_all")`, reset to its
  `presentation_idx`, and step through `decode_path(path)` (reuse
  `envs.utils.decode_action`), **recording `state.x` after every move** (extends
  `replay_packed_path`, which currently keeps only the final state).
- For each recorded state: `env_state_to_strs` → `canon.canonical_pair_nj` →
  the canonical `xXyY` pair. Emit one CSV row per solved presentation in the **greedy
  CSV schema**:
  `r1, r2, Nodes Visited, Path Length, Path`
  where `Path` is the flat list `[r1₀,r2₀,r1₁,r2₁,…]` of canonical states (init →
  trivial), `Path Length` = number of moves, `Nodes Visited` = `-1` (n/a for beam; kept
  for schema parity). Write `data/beam_paths.csv`.
- **Validation gates (hard — stop and fix if any fail):**
  1. Every replayed path reaches the trivial presentation `(x, y)` in exactly
     `path_length` steps (this is the repo's `check_paths` test; reuse it).
  2. Each consecutive on-path pair differs by exactly **one** substitution as a
     multiset (the PREP-A integrity check from `eda.ipynb`), since the modified relator
     can land in either slot.
  3. `len(Path) == 2*(path_length+1)`.
  Drop and **log** any path that fails (don't silently skip).
- Sanity report: for presentations greedy also solved, tabulate
  `beam_len − greedy_len` (how often / how much beam is shorter, equal, longer); count
  **new solves** among greedy-unsolved rows. (Descriptive — not a claim beam dominates.)

## Phase 4 — Merge into the min-d-o-t label table

**New script: `scripts/build_label_table.py`**

- Re-derive greedy on-path states from the CSV (the `on_path_states` construction from
  `eda.ipynb` PREP-B) and the beam on-path states from `data/beam_paths.csv`. For each
  state compute `remaining_dot = path_length − step_index` and `canon_key`.
- **Union** both sources, group by `canon_key`, take **`dot_min = min(remaining_dot)`**
  across all paths/sources (the min-aggregation motivated by EDA P0-3: within greedy
  data there was zero disagreement, so min-aggregation only starts to *bite* once the
  shorter beam paths are added — exactly this step).
- Write `data/dot_labels.parquet`:
  `canon_key, r1_canon, r2_canon, total_len, dot_min, dot_greedy, dot_beam, n_paths,
  source_min`. This is the supervised target set.
- **Validation:** `dot_min ≤ dot_greedy` wherever both exist; report how many keys had
  their label improved by beam and by how much; assert no key maps to two different
  `(r1_canon, r2_canon)`.
