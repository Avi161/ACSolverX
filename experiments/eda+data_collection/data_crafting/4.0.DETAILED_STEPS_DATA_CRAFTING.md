# Detailed Steps — Data Crafting for the d-o-t Regressor

*Execution decomposition of* `3.DATA_DISTRIBUTION_PLAN.md`*. That doc is the **decision** layer
(what & why); this is the **build** layer (ordered, atomic, verifiable steps). No new decisions are
made here — every knob points back to a* `3.` section. Where this doc and the source disagree, a
`SUPERSEDES` note says so explicitly.*

**Goal:** produce a training distribution in which a supervised model learns d-o-t (distance until
trivialization) from **structure, not relator length**, with a frozen proven-open eval set to score
hardness recognition honestly.

**Status:** planning artifact. All code described here is `[unverified]` until its step's acceptance
check runs. Everything in this doc is **CPU-only, no JAX** (canonicalization is `numba`-optional).

**Progress (checkbox legend** `[ ]` **pending ·** `[x]` **done & verified ·** `[x][-]` **done but the step was
changed — see the inline from→to note):** Phase 0 + Phase 1 are ✅ executed & verified
(commits `b46bac6`, `7665a89`); Phases 2–9 are pending.

**Originals are never modified.** Every output is a `*_v2` / new file. `dot_archive.jsonl`,
`dot_splits.npz`, `merged_best_paths.jsonl`, and all `scripts/`* keep their current behavior; new
work is additive (`make_splits_v2` next to `make_splits`, etc.).

## §A. Config & constants (single source of truth)

A new module `scripts/lib/dot_config.py` holds every tunable so the three build scripts and the report
notebook import one definition (no copy-paste drift). Values below are the §3-doc decisions.


| name                         | value                                                            | source                                  |
| ---------------------------- | ---------------------------------------------------------------- | --------------------------------------- |
| `L` (per-relator max_length) | `24`                                                             | env invariant; `canon` default          |
| `SEED`                       | `0`                                                              | frozen splits                           |
| `TARGET_SHARES`              | `{easy: .25, valley: .35, hard_solved: .15, hard_unsolved: .25}` | 3-§2                                    |
| `BAND_EDGES` (d-o-t)         | easy `0–10`, valley `11–50`, hard_solved `51+`                   | 3-§2                                    |
| `WEIGHT_CLIP`                | `(0.25, 5.0)`                                                    | 3-§2                                    |
| `TAIL_MULT`                  | `dot > 100 → ×0.5`                                               | 3-§2                                    |
| `SHORT_HARD_MULT`            | `×3` (capped at 5.0)                                             | 3-§2                                    |
| `AK_MIN, AK_MAX`             | `3, 8`                                                           | 3-§3a (cap = AK(8), headroom rationale) |
| `DIVERSITY_STRIDE k`         | default `2` (easy band only)                                     | 3-§2 diversity                          |
| `PER_PATH_CAP C`             | default `40`                                                     | 3-§2 diversity                          |
| `B_soft, B_hard`             | **re-derived** in Phase 0 (assert ≈ `48` / `150`)                | 3-§4                                    |


`B_soft`/`B_hard` are NOT hardcoded — Phase 0.2 computes them from the *current* solved-d-o-t
percentiles and **asserts** they round to the documented `p90≈48` / `p99≈150`. If a future data
change moves them, the assert fires loudly instead of silently using stale numbers.

**Encoding reminder (from** `scripts/lib/canon.py`**):** strings use letters `x X y Y` (`X`=x⁻¹, `Y`=y⁻¹);
env int8 literal uses `x→1, X→-1, y→2, Y→-2`, zero-padded to `2·L = 48`; canonical alphabet order is
`Y < y < X < x`. The cross-source key is `canon_key(r1, r2) → ("<r1>|<r2>", total_len)`.

## Phase 3 — Diversity / decorrelation  *(*`build_training_archive.py`*, part 2)*

The labelled archive is **path-derived**: consecutive on-path classes are ~1 S-move apart
(near-duplicates). This thins dense chains so the model sees varied regions, not clustered look-alikes
(3-§2 diversity). **Critical:** thin only the *dense easy band* — the rare high-d-o-t states are the
valley/hard signal (3-principle 3) and the `hard_test` pool, so they are exempt.

- [ ] **3.1 — Reconstruct path → class sequences.**

- *Action:* read `data/derived/paths/merged_best_paths.jsonl`; for each row with `best_path != []`, walk its state
trajectory `[r1_0,r2_0, …, r1_N,r2_N]`, and for each state apply the **same** `canon_key` used by
`build_dot_archive` → an ordered list of canonical keys per source path.
- *Inputs:* `data/derived/paths/merged_best_paths.jsonl`, `scripts/lib/canon.py`.
- *Why same key:* keep-set keys must be identical strings to archive keys, or the join silently drops
everything (3-arch-decision 1).
- *Acceptance:* every reconstructed key exists in the v2 archive (1:1 join, 0 orphans).

- [ ] **3.2 — Stride + cap, easy band only.**

- *Action:* per source path, keep every `k`-th state **whose d-o-t ≤ 10 (easy band)**; cap the kept
easy-band states at `C` per path. **Valley/hard_solved states (d-o-t ≥ 11) are kept unconditionally**
(or lighter stride if later needed).
- *Acceptance:* dropped set ⊆ easy-band labelled classes; no d-o-t ≥ 11 class dropped.

- [ ] **3.3 — Compute the keep-set (union, with hard exemptions).**

- *Action:* a labelled class survives if kept by **any** path. **Never dropped:** all `censored`
rows, all `tier=="named"` anchors, the probe look-alike key `canon_key("YXyxYx","xxxYYYY")`, and
any `group=="short_hard"` key. Drop the remaining un-kept easy-band labelled classes.
- *Output:* in-memory surviving row set (still pre-weight).
- *Acceptance:* `survivors < raw labelled`; censored/anchor/probe/short_hard all present; the
look-alike key present.

- [ ] **3.4 — Record diversity diagnostics.**

- *Action:* per surviving class store `n_paths` (how many source paths touched it) and a
`near_neighbour_count` proxy (count of surviving classes within 1 letter total-length and sharing a
relator — cheap structural proxy, exact ≤5-AC-move dedup is intractable per 3-§2).
- *Output:* fields carried to Phase 6 schema for the §8 spread report.
- *Acceptance:* diagnostics present on every surviving labelled row.

## Phase 4 — Loss metadata  *(*`build_training_archive.py`*, part 3)*

- [ ] **4.1 —** `loss_type`**.**

- *Action:* `regression` for labelled (`censored==false`); `hinge` for `censored==true` (this covers
both generic censored rows and the named anchors).
- *Acceptance:* every row has `loss_type ∈ {regression, hinge}`; all `hinge` rows are `censored`.

- [ ] **4.2 —** `bound_B`**.**

- *Action:* `bound_B = B_hard` for `tier=="named"` (believed non-trivializable); `bound_B = B_soft`
for all other censored rows (might be solvable, just not found). Labelled rows: `bound_B = null`.
- *Acceptance:* named → `B_hard`; generic censored → `B_soft`; labelled → `null`.

- [ ] **4.3 — (Documentation only) asymmetric solved loss is MODEL-side.**

- *Note:* labels are **upper bounds** (shortest path *found*), so the solved-row loss must penalize
*under*-prediction lighter than over-prediction, and eval must lead with Spearman (3-§4/§5/§8).
That lives in the **model plan**, not the data. The data's job here is only to **carry**
`tier`/`loss_type`/`bound_B` so the model can apply the right loss per row. **No fake numeric
labels are ever written for censored rows** (their `min_dot` stays `null`).

## Phase 5 — Bands & weights  *(*`build_training_archive.py`*, part 4 — order is load-bearing)*

Rebalance **loss mass** to the §2 target mixture by per-row weight (never resample — 3-principle 1).
Computed on the **post-diversity surviving set** so weights reflect what actually trains.

- [ ] **5.1 — Assign region.**

- *Action:* labelled → easy/valley/hard_solved by `BAND_EDGES`; censored + named → `hard_unsolved`.
- *Acceptance:* every row in exactly one of the 4 regions.

- [ ] **5.2 — Actual band shares.**

- *Action:* fraction of (count) rows in each region on the surviving set.
- *Acceptance:* the four shares sum to 1.0.

- [ ] **5.3 — Base weight.**

- *Action:* `w = clip(TARGET_SHARES[region] / actual_share[region], 0.25, 5.0)`.

- [ ] **5.4 — Multipliers.**

- *Action:* `if dot != null and dot > 100: w *= 0.5` (loose far-tail labels). `if group=="short_hard" or tier=="named": w *= 3`, then `w = min(w, 5.0)`.

- [ ] **5.5 — Normalize ONCE, last.**

- *Action:* `w *= len(rows) / sum(w)` so `mean(w) == 1` — applied **after** 5.4, not before. (Doing
it before the multipliers would break both the mean-1 invariant and the loss-mass target.)
- *Output:* per-row `weight`.
- *Acceptance:* `mean(weight) ≈ 1.0` (±0.01); per-band **effective loss mass** (`Σ weight` in band /
`Σ weight` total) ≈ `TARGET_SHARES` **within a tolerance** (the ×0.5/×3 multipliers perturb the
exact targets slightly — this is a tolerance band, not equality).

- [ ] **7.1 —** `counterexample_eval` **fold =** `tier=="named"` **ONLY (frozen, never trained).**

- *Action:* the 6 AK(n) + 2 Length-14 named anchors. **SUPERSEDES** source 3-§5, which froze

- [ ] **7.2 —** `hard_test` **fold = top 5% solved by d-o-t (frozen).**

- *Caveat (carry into the report):* "top 5% by d-o-t" = top 5% by *known upper bound* → may include
loose labels (recorded 150, true 50). Phase 8 + the model plan therefore lead with Spearman on this
fold, not MAE (3-§5).
- *Acceptance:* fold size ≈ 5% of labelled; disjoint from train/val/test and counterexample_eval.

- [ ] **7.3 —** `train / val / test` **= 80/10/10 of the remainder.**

- *Action:* remainder = everything except the two frozen folds. **Censored rows (incl. the 8
cousins) go into train + val/test** so the hinge is both trained and measured.
- *Acceptance:* fractions hold; censored present in train, val, and test.

- [ ] **7.4 — Probe assertion.**

- *Acceptance:* look-alike **in train, NOT in eval/hard_test**; AK(3) in `counterexample_eval`. (We
assert membership/exclusion, not a specific random index a seeded shuffle can't guarantee.)

- [ ] **7.5 — Leakage & determinism.**

- *Acceptance:* no canonical key appears in two folds; rerun with `SEED` → byte-identical `.npz`.
