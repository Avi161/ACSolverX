## 1. Executive summary

**Goal.** Produce a training distribution in which a supervised model learns **d-o-t** (distance until
trivialization) from **structure, not relator length**, with a frozen proven-open eval set to score
hardness recognition honestly.

**What shipped.** A frozen, seed-pinned split file (`dot_splits_v2.npz`) the model plan can load directly,
plus an audited acceptance report. The pipeline runs end-to-end on CPU in seconds and is fully
deterministic and idempotent:

```
dot_config (constants) → build_anchors (P1) → build_training_archive (P2–6) →
dot_dataset.emit_splits_v2 (P7) → build_v2_report (P8) → ak_trap_set handoff (P9)
```

**Headline numbers (v2).** 38,384 rows = **33,422 solved** (regression) + **4,954 generic censored**
(hinge) + **8 named anchors** (hinge, frozen eval). Five frozen folds: **train 29,365 / val 3,670 /
test 3,670 / hard_test 1,671 / counterexample_eval 8**. Whole-archive band loss-mass reproduces the §2
target: **easy .256 / valley .358 / hard_solved .128 / hard_unsolved .257**, `mean(weight)=1.0000`.

**The honest framing (set by the dual reviewers).** The 8-row `counterexample_eval` is **all-hard with no
ground-truth d-o-t → no MAE**; a constant-output model passes it. It is a **length-stratified
separation/ranking** anchor: under v1's single len-13-14 baseline only `{AK(3), Length-14 ×2}` (len ≤14,
n=3) are length-matched; the longer anchors gain a **spurious separation that grows with length**
(`solved_median_dot(L)` rises monotonically, so AK(8)@23 is the *most* length-gameable — the fix is
per-length-matched baselines). AK(3)'s real confound is **r1-cousin leakage**. The statistically-powered
hardness metric is **Spearman on `hard_test`** (1,671 real labels). `bound_B` (48/150) is a **heuristic
floor, never a lower bound**. See §5.

## 5. Honesty caveats carried forward (dual-reviewer findings)

These are **not** defects — they are the limits of v1, surfaced so the model plan and any paper claim
stay honest. They live in `v2_report.json["caveats"]` and the notebook §8.

1. **`counterexample_eval` has no MAE-able metric.** All-hard, no ground truth; a constant model passes
   it. Report it as **separation/ranking** (`P(eval) ≫ P(short-easy)`), never a bare value.
2. **AK(3)'s dominant confound is r1-cousin leakage** (lead with this, not length). AK(3) is
   **length-matched** to the baseline (spurious separation ≈0), so essentially all of its separation must
   come from structure — or from the **116 train rows sharing its exact r1 `YXYxyx`** (incl. 3 short_hard
   cousins; by the Havas–Ramsay length-≤13 dichotomy *plausibly the same AC-problem*, AC-equivalence being
   undecidable). A "pass" can be **r1-substring memorization**, not de-novo recognition. *Per-anchor, not a
   global ordering* — the long anchors flip to a length confound (next).
3. **The length confound (mechanism, corrected).** The eval is **separation vs a baseline**, with no
   ground truth, so a length-exploiting regression head predicts ≈ `solved_median_dot(L)`, which **rises
   monotonically** with length (13→6 … 23→23). Against v1's single len-13-14 baseline this buys a spurious
   separation that **grows with length** — so **AK(6/7/8)@19/21/23 are the *most* length-gameable** (+9/+12/
   +17), not the least. The len ≤14 anchors are *length-matched-to-the-v1-baseline*; len ≥15 *need a
   per-length baseline*. **Do not call the long anchors "clean" under the current metric.** The constructive
   fix (model plan): **per-length-matched baselines** `P(AK(n)@L) ≫ P(solved@L)` — the data has thousands
   of solved rows at every anchor length (15:2735, 17:8603, 19:2467, 21:2571, 23:2120) — which removes the
   length confound from all 8 anchors, leaving AK(3)'s r1-leakage as the only residual.
4. **Headline the TRAIN cut, not the archive.** Train's `hard_solved` mass is **doubly depleted** to
   ~.087 (top-5% frozen into `hard_test` + the ×0.5 far-tail discount); the .256/.358/.128/.257 is the
   Phase-5 reproduction, not training reality.
5. **Weight balances per-band WEIGHT mass, not loss/gradient mass** — hinge (censored) and regression
   (labelled) are different scales, so equal weight ≠ equal gradient.
6. **The Phase-3 drop is anti-diverse** (68.5% single-path); information-preserving reweighting would
   suffice. The real easy-mass control is Phase-5 reweighting + the by-class split, not the drop.
7. **`hard_test` membership is upper-bound-contaminated + heteroscedastic** (loose-label classes
   populate the tail) → lead with **Spearman**, not MAE.
8. **`bound_B` is a heuristic floor, never a lower bound.** `B_hard=150` rests on the *solved* p99, has no
   group-theory backing, and can err in **both** directions. **All 8 named anchors — the 6 AK(n) *and* the
   2 Length-14** (which fall **outside** the Havas–Ramsay length-≤13 dichotomy, so they could be a genuinely
   new hard class *or* solvable-but-missed) — are *potential* counterexamples / believed-hard, **never
   proven non-trivializable**. Cousins are *suspected*, never asserted AC-equivalent to AK(3).
9. **`short_hard`+`named` = 16 rows ≈ 0.21% of loss mass** → the `×3` is numerically near-inert. v1 does
   **not** materially fund the look-alike-trap goal; the real fix (automorphic short-hard data) is deferred.
10. **Band-boundary mis-binning** of loose (50,100] labels (the tail ×0.5 only fires >100); mitigation is
    the model-side asymmetric loss + Spearman. d-o-t is treated as a **class invariant** under the
    rotation/inverse/swap-equivariant, length-capped action space (Phase-3.1 evidence: 1-S-move neighbours
    + 0-mismatch `min_dot` re-walk).

## 6. Next steps

**Model planning (`experiments/training/1.PLAN.md`).**
- Loss: **Huber on `log1p(d-o-t)`** for labelled (`regression`) + **one-sided hinge(`bound_B`)** for
  censored (`hinge`), weighted per the `weight` column; labels are upper bounds → penalize
  *under*-prediction lighter than over (asymmetric solved loss is MODEL-side, 4.0 §4.3).
- Eval: **lead the hardness section with Spearman on `hard_test`**; present `counterexample_eval` as a
  **per-anchor contrast/anchor** (the AK(3)-vs-look-alike pair is an n=1 illustrative probe), never a single
  number. Report MAE/Spearman on `test`, `hard_test`, and the named anchor separately.
- **Compute per-length-matched short-easy baselines** (`v2_report.json` carries `per_length_solved` for
  every anchor length) so the eval is `P(AK(n)@L) ≫ P(solved@L)` — this **subtracts the monotone
  `solved_median_dot(L)` length confound** and makes all 8 anchors usable (under v1's single len-13-14
  baseline only n=3 are length-matched, and AK(8)@23 buys the largest spurious separation).
- Optional auxiliary solvable-within-budget head.

**Deferred data work (next build).**
- **Automorphic short-hard data** — the real fix for the look-alike trap (16 rows ≈ 0.21% is near-inert).
- **Length-conditioned `B_soft(L)`** (3-§4a) — the flat `B_soft=48` understates len-17 unsolved, and after
  pinning the 8 cousins to train the held-out hinge is validated **only on long censored against a flat
  48**; `B_soft(L)` closes that gap.
- **Per-row search-context enrichment** (`best_len_reached`, budget/horizon) — a deferred schema change;
  `merged_best_paths` carries no best-length-reached for *unsolved* rows yet.

**Search-time handoff (already wired).** `ak_trap_set.json` is consumed by `experiments/PLAN.md §8`
(known-AK(n) + novel-basin discovery → `discovered_minima.jsonl`, living-set promotion) and
`experiments/training/1.PLAN.md §8` (AK(n) trap-rate metric). **Consumer-side gate (named for whoever
implements the monitor):** canonicalize each expanded state via `canon.canon_key`, **never** the env's
per-move Booth rotation (not a canonical form) — that is the one place the handoff would silently break,
and the env→`canon_key` round-trip on AK(3) is now asserted in `test_acceptance_v2.py`. **Membership
semantics:** a hit means the search *landed on* a known hard presentation; the set is **closed-world
(16 keys)**, so a transit-*through* detector must not over-fire on intermediate states — match on the
canonical key of a settled/expanded state, not every node on the path. The set is **living**: recurring
novel basins are promoted back into `ak_trap_set.json` (and as new censored anchors) on the next data
build — closed-world for known basins, open-world for new ones.
