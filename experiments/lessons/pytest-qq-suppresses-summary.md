# [2026-07-09] `-q` in pytest.ini addopts + `-q` on the command line = `-qq` = no summary [TRAP]
`pytest.ini` had `addopts = --basetemp=.pytest_tmp -q` and the documented command is `pytest experiments/greedy_tests -q`. pytest sums verbosity flags, so the pair becomes `-qq`, which **suppresses the final `N passed, M skipped` line** — the run still exits 0, but reports no counts, and a grep for "passed" in the log finds nothing. Spent a round chasing the missing summary before spotting it. **Rule:** never put `-q`/`-v` in `addopts` if the documented invocation also passes one; put only non-verbosity flags (`--basetemp`, `--strict-markers`) there. And when a log shows dots but no summary, suspect doubled verbosity flags before suspecting a crash.
