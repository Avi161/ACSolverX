# [2026-07-08] W&B auto-auth: needs a Colab Secret; getpass paste was malformed [TRAP]
User reported "auto authenticate not working — still prompts." Two real causes: (1) `AUTO_AUTHENTICATE_WANDB=True` can only be PROMPTLESS if a **Colab Secret** named `WANDB_API_KEY` exists (with Notebook-access ON) — there is no other persistent key store in Colab (`~/.netrc` and `os.environ` are cleared on runtime restart). With no secret it falls back to `getpass`, which asks once per fresh runtime.
