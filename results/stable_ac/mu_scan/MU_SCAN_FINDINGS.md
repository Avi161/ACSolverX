Generated by `experiments/stable_ac/cov/mu_descent_scan.py` (cap 24, keep 10) from the aca_124 class reps; data `mu_scan_aca124_d2_k10_mrl24.jsonl`. μ = Aut(F₂)-minimal total length of an orbit (`aut_canon`). ## Findings

1. **19 of the 124 classes have a strictly descending orbit-floor path within two gated subword-CoV hops — 15 of them hop-2-ONLY.** The hop-2-only descents climb first, then drop below the input floor: aca_97/98 go μ 24 → 27 (hop 1) → **19** (hop 2); aca_81/85: 22 → **18**; aca_58/123: 20 → **17**; aca_34/122: 18 → **16**; aca_120/121: 16 → **15**; aca_72: 20 → 18; aca_95: 21 → 19; aca_43/44: 19 → 18; aca_88: 23 → 21; plus the four hop-1 descenders (aca_99/100: 25 → 22, aca_105/106: 25 → 24). This is the uphill-then-downhill shape the floor refutation predicted — and it is exactly what a per-hop-μ-pruning tree would have missed.
2. **The 50k sweep never searched any of these.** It searched hop-1 outputs only; the descended hop-2 orbits are virgin territory for the greedy. The 28 concrete descended start pairs (every chain re-derived and orbit-μ-verified) are exported as **`data/ms_unsolved_reps/mu_descents_d2.csv`** (`name,r1,r2,source_class,mu_source,mu,hops,z_path`), loadable as bench `mu_descents_d2` in the portfolio runner.
3. **Certificate accounting**: each exported start is reached from its class rep by 1–2 CoV hops (stable supermoves), so a greedy solve from it certifies **stable AC-triviality** of the whole source class — segmented certificate `[CoV chain] + [greedy path]`, same convention as the restart tree. A solve here would remove one of the 124 from the stable ledger.
4. Reach growth: 919 distinct orbits at hop 1 → **4,772** cumulative at hop 2 (~5× per hop, no saturation), consistent with the shipped geometric-growth remark.
5. Scope: `keep = 10` lowest-μ hop-1 orbits expanded per class, cap 24 — a wider `keep` can only ADD descents; these 19 are a lower bound.
6. **Cap-48 arm: exact null.** The full cap-48 re-scan (`mu_scan_aca124_d2_k10_mrl48.jsonl`) reproduces the cap-24 map identically — same 19 descending classes, same best floors, same 919/4,772 orbit reach. At these input lengths the 24-cap rejects no depth-2 CoV candidate that matters; the high-cap hypothesis (user's T9 push) is not falsified but its payoff, if any, lives deeper in the iterated tree or inside the 50k searches (watch `max_popped_total` in the stall-escape rows), not at tree depth 2.
