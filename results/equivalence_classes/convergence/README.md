# `convergence/` — the corroborating sweeps

**This is not an archive.** These four sweeps have zero code references, but they are *cited
evidence*: they are the other rows of the five-configuration table in
[`../EQUIVALENCE_FINDING.md`](../EQUIVALENCE_FINDING.md) §3, and that table is the argument for
believing the shipped number.

The claim is not "one sweep found 126 classes." It is "**every configuration we varied lands in the
same place**" — a different move set, a different cap, extra seed sources. Delete these and the
headline result loses its support.

| sweep | what it varies | classes | what it shows |
|---|---|---|---|
| `sweep_seam_26_30.json` + `classes_sweep_seam_26_30.csv` | cap 26, 30 pops/source | 127 | the cheap baseline config |
| `sweep_full_26_60.json` | `full` move set — every `(k1,k2)`, not just cancelling seams | 127 | **the enlarged move set buys nothing** |
| `sweep_seam_26_120_ms.json` | + the 550 raw Miller–Schupp presentations seeded as bridge sources | 127 | **raw MS cells do not bridge classes** |
| `sweep_seam_26_120_j.json` | + the 783 states the 1M-node greedy sweep recorded, seeded too | 127 | extra seeds do not bridge either |

The canonical sweep (`../sweep/sweep_seam_28_250.json`, cap 28, 250 pops) is the tighter one: **126**. The
four above converge on 127; raising the cap merges one more pair. Counts are upper bounds — the
search is sound and incomplete, so **every merge found is a proof and no merge found proves nothing.**

Run logs for three of these are in [`../logs/`](../logs/). `run_seam_26_30.log` was never captured —
a cosmetic gap, not a data gap.

## ⚠ Schema drift

These JSONs were generated by an **earlier version** of `run_sweep.py` (in a since-deleted worktree —
the absolute paths baked into the logs still show it). They use the key `n_classes_excl_trivial` and
lack the top-level `rep_classes` / `pre_union` keys. The `run_sweep.py` **currently in the tree**
writes `n_classes` and always emits both.

Practical effect: re-running the "reproduce from scratch" recipe in [`../README.md`](../README.md)
will **not** byte-match the checked-in JSON. `make_class_table.py` tolerates it
(`data.get("rep_classes")` with a reconstructing fallback); nothing else in the pipeline was checked
against the old schema. The *numbers* are unaffected — only the container.

A fresh `run_sweep.py 26 30 seam` writes to the **parent** directory, not here. This folder holds the
committed historical copies.
