# [2026-07-09] Comprehensive test suite: three layers so one bug can't hide another [WORKS]
Built `experiments/greedy_tests/` (see its README; the mandatory-run rules are at the top of this file). The design decision that matters: **do not write an oracle that reproduces `nodes_explored`.** Such an oracle must replicate priority + enumeration order + comparator + closed-set + cap filter bit-for-bit, which makes it a near-copy of the implementation (shared bugs), and it would wrongly reject a *correct* future stable-AC solver that explores in a different order. Instead the oracle's jobs were split three ways:
1. **`spec/` вЂ” a general-`n` reference for the math only** (words as tuples of signed ints, so `n_gen` is free). Written from the definitions: brute-force minimal rotation instead of Booth, reduction from the definition. Agreement is then evidence, not a restatement. Verified: reduce + canonical agree exhaustively over all 5,461 words of length в‰¤ 6 in both cyclic modes; canonical neighbour sets agree on 48 presentations Г— 2 modes.
2. **Abelianization = the independent oracle.** `M[i][g]` = exponent sum. A substitution move is `row_i в†ђ row_i + sВ·row_j` (an elementary row op в‡’ `det` exactly preserved); canonicalisation only negates/permutes rows в‡’ `|det|` preserved; AC4 borders `M` with a unit row/col в‡’ preserved. Trivial presentation has `M = I`, so **every AC-trivial presentation has `|det| = 1`**. Confirmed on all 640 `ms640_solved` (SNF = I too) and all 261 unsolved reps, and on every neighbour the real numba expander emits. The solver never computes an exponent sum, so this catches design errors the spec could share.
3. **`adapters.py` = the forward-compat seam.** `SolverAdapter.supports(pres)` + normalised `SearchStats`; contract tests iterate `ALL_ADAPTERS` Г— fixtures and skip unsupported pairs. The `SpecAdapter` already runs at `n_gen=3`, so the stable-AC tests are live coverage, not dead skips.
**Surprise worth keeping:** the spec search *does* reproduce `nodes_explored` exactly at `n_gen=2` (0 mismatches), because the heap key `(total, depth, key)` is a strict total order вЂ” pop order cannot depend on push order, and `depth` is fixed by the enumeration order. It is pinned as a *bonus* check on the two 2-generator solvers and explicitly NOT as a contract.
